## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 야구 선수 능력 분석 및 5툴 플레이어 판정 시스템 구현으로, 사용자로부터 선수 데이터를 입력받아 리스트에 저장하고, 조건 기반 분석을 통해 결과를 도출하는 시스템을 구축하는 것입니다. 특히 if-elif 조건문과 and 논리 연산자를 활용하여 선수의 능력에 따른 등급과 특별 칭호를 부여하는 로직을 구현하는 데 목적이 있습니다.
  * input() 함수를 활용하여 선수의 이름, 타율, 도루 개수, 수비율, 송구 속도, 장타율을 입력받아 하나의 리스트로 저장합니다.
  * 이후 조건문을 활용하여 타율, 도루, 수비율, 송구 속도, 장타율이 모두 기준값 이상일 경우 5툴 플레이어로 판정하며, 해당 조건을 만족하지 못할 경우에는 추가적인 조건을 적용하여 호타준족 또는 수비 장인과 같은 특별 칭호를 부여합니다.
  * 또한 반복문과 리스트를 활용하여 여러 선수의 데이터를 처리하고, max() 함수를 이용해 가장 높은 타율을 가진 선수를 타격왕으로 선정하는 기능을 구현하였습니다.
  * 이와 같은 과정을 통해 입력된 데이터를 체계적으로 저장하고, 논리 연산을 기반으로 선수의 능력을 분석하여 결과를 도출하는 프로그램을 완성하였습니다.
    
### **🤖 AI 파트너십 과정** 
1. **내용 1: 선수 능력 분석 로직 설계**
    * **프롬프트 요약:** "야구 선수 데이터를 활용해 1학년 수준에서 리스트와 조건문으로 5툴 플레이어를 판정하는 프로그램을 만들고 싶어. 현실적인 기준과 조건을 반영한 판정 로직을 제안해줘"
    * **적용 내용:** AI가 제안한 다중 조건 기반 판정 구조를 바탕으로, 타율·도루·수비율·송구 속도·장타율의 기준값을 설정하고, 이를 모두 만족할 경우 5툴 플레이어로 판정하는 로직을 구현함. 또한, 일부 조건만 만족할 경우 ‘호타준족’, ‘수비 장인’과 같은 추가 칭호를 부여하는 복합 조건 구조를 코드에 반영함.

2. **내용 2: 리스트 기반 데이터 처리 구조 설계**
    * **프롬프트 요약:** "파이썬 기초 문법(리스트, 반복문, 조건문)을 활용해 여러 선수 데이터를 효율적으로 저장하고 분석하는 프로그램 구조를 단계적으로 설계해줘."
    * **적용 내용:** AI의 제안을 바탕으로 ‘선수 데이터 입력 및 리스트 저장 → 반복문을 통한 전체 데이터 순회 → 조건문을 활용한 선수 능력 판정 → 최고 타율 선수(타격왕) 선정’으로 이어지는 구조를 설계함. 이를 통해 단순 입력을 넘어 데이터 저장–처리–분석으로 이어지는 기본적인 데이터 처리 흐름을 구현함.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 리스트 데이터 접근 시 혼동 **
    * **원인:** 선수 데이터를 리스트로 저장하면서 각 요소의 위치(인덱스)를 혼동하여 잘못된 값을 참조할 가능성이 있었음.
    * **해결:** [이름, 타율, 도루, 수비율, 송구속도, 장타율] 순서를 명확히 정리하고, p[0], p[1], p[2]와 같이 각 인덱스의 의미를 구분하여 사용함.
2. **문제 2: 조건 우선순위 오류 **
    * **원인:** 5툴 플레이어 조건과 추가 칭호 조건을 동시에 검사하면서,5툴 플레이어임에도 불구하고 다른 칭호가 부여될 가능성이 있었음.
    * **해결:** if (5툴 조건) → else → 추가 조건 구조로 조건문의 우선순위를 재구성하여 5툴 플레이어가 최우선으로 판정되도록 수정함.

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정
### **✨3차 과제 업데이트 내용:**
  * 이번 과제에서는 기존 2차 과제에서 구현한 KBO 타자 5툴 플레이어 판정 시스템을 함수 중심 구조로 개선하고, 메뉴 기반 상호작용 시스템으로 확장하였다. 기존 프로그램은 선수 데이터를 입력받아 조건문을 통해 5툴 플레이어 여부를 판정하는 단순 실행 구조였다. 이번 3차 과제에서는 while True 반복문을 활용한 메인 메뉴 시스템을 추가하여 사용자가 원하는 기능을 선택적으로 실행할 수 있도록 구현하였다. 프로그램 실행 시 다음과 같은 메뉴가 출력된다.
1.선수 등록, 2.전체 선수 조회, 3.선수 분석, 4.프로그램 종료
또한 기존 기능을 개별 함수로 분리하여 코드 구조를 체계화하였다.
add_player() : 선수 정보 입력, show_players() : 전체 선수 목록 조회, analyze_player(player) : 선수 능력 분석, calculate_score(player) : 종합 점수 계산, main_menu() : 메뉴 제어
특히 함수 간 매개변수 전달과 return 반환값을 적극적으로 활용하여 데이터 처리 과정을 명확히 구성하였다. 추가적으로 전역 변수인 players 리스트를 선언하고, 선수 등록 함수 내부에서 global 키워드를 사용하여 데이터 저장 기능을 구현하였다. 이를 통해 단순 조건 판정 프로그램에서 벗어나, 반복 실행 가능한 관리형 데이터 분석 시스템으로 발전시켰다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  기존의 순차 실행형 선수 분석 프로그램을 3차 과제 조건에 맞춰 while 반복문과 메뉴 선택 구조로 발전시키려면 어떤 방식으로 설계하면 좋을지 설명해줘.
    * **적용 내용:** AI의 설명을 통해 순차적으로 한 번만 실행되던 기존 프로그램을 반복적으로 사용할 수 있도록 메뉴 기반 제어 구조의 개념을 이해하였다.
이를 바탕으로 프로그램 시작 후 사용자가 원하는 기능을 선택하고, 종료 전까지 반복 실행되는 흐름으로 전체 구조를 재설계하였다.
 2. **내용 2**
    * **프롬프트 요약:**  파이썬 기초 문법 범위에서 함수 분리, 매개변수, 반환값, 전역 변수의 역할을 어떻게 구분하여 프로그램을 구조화할 수 있는지 설명해줘.
    * **적용 내용:** AI의 설명을 참고하여 입력, 조회, 분석 기능을 각각 독립된 함수로 분리하는 방향을 설정하였다.
또한 함수 간 데이터 전달을 위해 매개변수와 반환값의 활용 원리를 이해하였고, 프로그램 전체에서 공통으로 사용하는 선수 리스트는 전역 변수로 관리하는 구조를 적용하였다.
이를 통해 코드의 역할 분담이 명확해지고 전체 흐름을 보다 체계적으로 구성할 수 있었다.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 무한루프종료불
     * **원인:** 초기 구현에서 종료 메뉴 선택 시 break가 올바르게 실행되지 않아 프로그램이 계속 반복되었음.
     * **해결:** 메뉴 선택 조건문에서 "4" 입력 시 break가 실행되도록 수정하여 정상 종료되도록 개선함.
  2. **문제 2:** 함수 내부 데이터 저장 오류
     * **원인:** 함수 내부에서 선수 리스트를 수정할 때 지역 변수로 인식되어 데이터가 저장되지 않았음.
     * **해결:** global players를 선언하여 전역 리스트를 수정할 수 있도록 해결함.

### 🟥 [4차 과제: V4.0] 모듈화 및 데이터 확장 (배열과 함수) - 🌟최종 완성 -- 향후 작성 예정
### **✨4차 과제 업데이트 내용:**
  * 내용.이번 과제에서는 기존에 구현한 KBO 타자 5툴 플레이어 판정 시스템에 데이터 관리 기능을 추가하여 프로그램의 완성도를 높였다. 여러 선수의 정보를 하나의 자료구조에 저장할 수 있도록 이중리스트를 활용하였으며, 선수 등록 시 새로운 데이터를 리스트에 누적 저장하도록 구현하였다. 또한 반복문을 이용해 저장된 모든 선수 정보를 표 형태로 출력할 수 있도록 개선하여 데이터 확인이 편리해졌다. 추가로 선수 데이터를 파일에 저장하는 기능을 구현하여 프로그램 종료 후에도 기록을 보관할 수 있도록 하였으며, 파일 입출력 과정에서 발생할 수 있는 오류와 잘못된 숫자 입력으로 인한 오류를 예외처리 구문으로 해결하여 프로그램이 중단되지 않도록 하였다. 이를 통해 단순히 선수를 판정하는 수준을 넘어 데이터를 저장하고 관리할 수 있는 형태의 선수 분석 프로그램으로 발전시켰다.

    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:**  기존의 순차 실행형 선수 분석 프로그램을 3차 과제 조건에 맞춰 while 반복문과 메뉴 선택 구조로 발전시키려면 어떤 방식으로 설계하면 좋을지 설명해줘.
    * **적용 내용:** AI의 설명을 통해 순차적으로 한 번만 실행되던 기존 프로그램을 반복적으로 사용할 수 있도록 메뉴 기반 제어 구조의 개념을 이해하였다.
이를 바탕으로 프로그램 시작 후 사용자가 원하는 기능을 선택하고, 종료 전까지 반복 실행되는 흐름으로 전체 구조를 재설계하였다.
 2. **내용 2**
    * **프롬프트 요약:**  파이썬 기초 문법 범위에서 함수 분리, 매개변수, 반환값, 전역 변수의 역할을 어떻게 구분하여 프로그램을 구조화할 수 있는지 설명해줘.
    * **적용 내용:** AI의 설명을 참고하여 입력, 조회, 분석 기능을 각각 독립된 함수로 분리하는 방향을 설정하였다.
또한 함수 간 데이터 전달을 위해 매개변수와 반환값의 활용 원리를 이해하였고, 프로그램 전체에서 공통으로 사용하는 선수 리스트는 전역 변수로 관리하는 구조를 적용하였다.
이를 통해 코드의 역할 분담이 명확해지고 전체 흐름을 보다 체계적으로 구성할 수 있었다.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** ...숫자입력오
     * **원인:** ...사용자가 타율이나 도루 개수를 입력할 때 문자를 입력하면 ValueError가 발생하여 프로그램이 종료되었음.
     * **해결:** ..try-except를 적용하여 잘못된 입력 시 오류 메시지를 출력하고 다시 입력받도록 수정함.
* 2. **문제 2:**파일불러오기오류
     * **원인:** 프로그램 실행 시 저장 파일이 존재하지 않으면 FileNotFoundError가 발생하였음.
     * **해결:** try-except를 이용하여 파일이 없을 경우 안내 메시지를 출력하고 빈 데이터 상태로 프로그램을 시작하도록 수정함.

# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* KBO 타자 5툴 플레이어 판정 시스템
<br>


## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 KBO 리그 타자의 주요 성적 데이터를 기반으로 선수의 종합적인 능력을 분석하는 시스템이다. 사용자는 타자의 한 시즌 타율, 장타율, 도루개수, 송구속도, 수비율이라는 5툴을 평가하는 타자 성적 데이터를 입력하게 된다. 입력된 데이터를 바탕으로 타격, 장타력, 주루, 수비, 송구 능력을 각각 한 개의 툴로 평가하여 모든 능력이 일정 기준 이상일 경우 ‘5툴 플레이어’로 판정한다. 추가적으로 5툴 플레이어가 아니더라도 특정 능력 조합이 뛰어난 경우 다양한 특별 칭호를 부여하여 선수의 특징을 보다 세밀하게 분석한다. 예를 들어 장타력과 주루 능력이 높은 경우 ‘호타준족’, 수비 능력만 뛰어난 경우 ‘수비 장인’과 같은 칭호를 부여한다. 여러 명의 선수 데이터를 리스트에 저장하여 전체 선수 기록을 관리할 수 있도록 설계한다. 사용자는 메뉴를 통해 선수 등록, 분석 결과 확인, 전체 선수 목록 조회 등의 기능을 사용할 수 있다. 최종적으로 이 프로그램은 데이터 기반으로 선수의 가치 정량적으로 평가하는 분석 도구로 완성될 예정이다.
<br>

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
  * 이번 과제는 야구 선수 능력 분석 및 5툴 플레이어 판정 시스템 구현으로, 사용자로부터 선수 데이터를 입력받아 리스트에 저장하고, 조건 기반 분석을 통해 결과를 도출하는 시스템을 구축하는 것입니다. 특히 if-elif 조건문과 and 논리 연산자를 활용하여 선수의 능력에 따른 등급과 특별 칭호를 부여하는 로직을 구현하는 데 목적이 있습니다.
  * input() 함수를 활용하여 선수의 이름, 타율, 도루 개수, 수비율, 송구 속도, 장타율을 입력받아 하나의 리스트로 저장합니다.
  * 이후 조건문을 활용하여 타율, 도루, 수비율, 송구 속도, 장타율이 모두 기준값 이상일 경우 5툴 플레이어로 판정하며, 해당 조건을 만족하지 못할 경우에는 추가적인 조건을 적용하여 호타준족 또는 수비 장인과 같은 특별 칭호를 부여합니다.
  * 또한 반복문과 리스트를 활용하여 여러 선수의 데이터를 처리하고, max() 함수를 이용해 가장 높은 타율을 가진 선수를 타격왕으로 선정하는 기능을 구현하였습니다.
  * 이와 같은 과정을 통해 입력된 데이터를 체계적으로 저장하고, 논리 연산을 기반으로 선수의 능력을 분석하여 결과를 도출하는 프로그램을 완성하였습니다.
    
### **🤖 AI 파트너십 과정** 
1. **내용 1: 선수 능력 분석 로직 설계**
    * **프롬프트 요약:** "야구 선수 데이터를 활용해 1학년 수준에서 리스트와 조건문으로 5툴 플레이어를 판정하는 프로그램을 만들고 싶어. 현실적인 기준과 조건을 반영한 판정 로직을 제안해줘"
    * **적용 내용:** AI가 제안한 다중 조건 기반 판정 구조를 바탕으로, 타율·도루·수비율·송구 속도·장타율의 기준값을 설정하고, 이를 모두 만족할 경우 5툴 플레이어로 판정하는 로직을 구현함. 또한, 일부 조건만 만족할 경우 ‘호타준족’, ‘수비 장인’과 같은 추가 칭호를 부여하는 복합 조건 구조를 코드에 반영함.

2. **내용 2: 리스트 기반 데이터 처리 구조 설계**
    * **프롬프트 요약:** "파이썬 기초 문법(리스트, 반복문, 조건문)을 활용해 여러 선수 데이터를 효율적으로 저장하고 분석하는 프로그램 구조를 단계적으로 설계해줘."
    * **적용 내용:** AI의 제안을 바탕으로 ‘선수 데이터 입력 및 리스트 저장 → 반복문을 통한 전체 데이터 순회 → 조건문을 활용한 선수 능력 판정 → 최고 타율 선수(타격왕) 선정’으로 이어지는 구조를 설계함. 이를 통해 단순 입력을 넘어 데이터 저장–처리–분석으로 이어지는 기본적인 데이터 처리 흐름을 구현함.
    
### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 리스트 데이터 접근 시 혼동 **
    * **원인:** 선수 데이터를 리스트로 저장하면서 각 요소의 위치(인덱스)를 혼동하여 잘못된 값을 참조할 가능성이 있었음.
    * **해결:** [이름, 타율, 도루, 수비율, 송구속도, 장타율] 순서를 명확히 정리하고, p[0], p[1], p[2]와 같이 각 인덱스의 의미를 구분하여 사용함.
2. **문제 2: 조건 우선순위 오류 **
    * **원인:** 5툴 플레이어 조건과 추가 칭호 조건을 동시에 검사하면서,5툴 플레이어임에도 불구하고 다른 칭호가 부여될 가능성이 있었음.
    * **해결:** if (5툴 조건) → else → 추가 조건 구조로 조건문의 우선순위를 재구성하여 5툴 플레이어가 최우선으로 판정되도록 수정함.
